# 무한매수법 V4.0 — 레퍼런스 스펙 (백테스트/개발용 내부 문서)

## Context

라오어(라오어 카페) **무한매수법 V4.0 일반모드 + 소진후 리버스모드**를 코드로 백테스트·개발하기 위한 정밀 레퍼런스.
원문(첨부 이미지 3종: 일반모드 §1~8, 리버스모드 §1~7)을 그대로 정리한 것이며, 백테스트 엔진과 자동주문 툴 구현의 **단일 진실원본(source of truth)**으로 사용한다.

> ⚠️ 저작권: 원문은 "재가공·편집하여 외부 공간(개인 홈페이지·유튜브 등) 제공 금지"를 명시. 이 문서는 **개인 백테스트/개발 용도 내부 문서**로만 사용하고 외부 배포하지 않는다.

대상 종목: **TQQQ**(3x 나스닥100), **SOXL**(3x 반도체). 그 외 비공식.
핵심 아이디어: 3배 레버리지의 변동성 끌림 위험을 **분할매수 + 분할매도 + 평단관리**로 통제하고, 보유 0이 되는 사이클을 무한 반복.

---

## 1. 상태 변수 (백테스트 엔진이 매일 들고 가야 할 값)

| 변수 | 의미 |
|---|---|
| `원금` | 사이클 시작 시 고정 자본. 외부 추가투입 금지. |
| `분할수 N` | 20 / 30 / 40 중 선택. 작을수록 공격적·위험. |
| `종목` | TQQQ 또는 SOXL (별% 공식·지정가% 분기) |
| `T` | Turn = 진행회차 (실수, 소수점 제한 없음) |
| `평단` | 보유분 평균단가 |
| `보유수량` | 현재 보유 주식 수 |
| `잔금` | 미투입 현금 (원금에서 매수에 안 쓴 잔액) |
| `모드` | `일반모드` / `리버스모드` |
| `1회매수금` | 그날 매수 시도 금액 (아래 §4 공식) |

사이클 = 보유 0(일반모드 종료)에서 다음 보유 0까지. 종료 시 §8 규칙으로 재시작.

---

## 2. T값 갱신 규칙 (V4 단순화 — 이전 버전과 다름)

`T` = Turn(진행회차). `1.0T` = 1회분 진행됨. 체결 이벤트마다 갱신:

| 이벤트 | T 갱신 |
|---|---|
| 1회분 전액 매수 체결 | `T += 1` |
| 절반 매수 체결 | `T += 0.5` |
| 쿼터매도 발생 | `T = 직전T × 0.75` |
| 지정가매도 후 같은날 LOC 매수 체결 | `T = 직전T×0.25 + 1` 또는 `직전T×0.25 + 0.5` |

소수점 제한 없음 (예: `T = 5.165479525`).
주: "1회분/절반"은 그날 매수 시도금 대비 **체결된 비율** 기준.

---

## 3. 별지점(★점) — 매수/매도 기준선 (V2.2부터, 예전과 동일)

별% 공식 (퍼센트):

| 종목 / 분할 | 별% |
|---|---|
| TQQQ 20분할 | `15 − 1.5·T` |
| TQQQ 40분할 | `15 − 0.75·T` |
| SOXL 20분할 | `20 − 2·T` |
| SOXL 40분할 | `20 − T` |

- **별지점 = 평단 × (1 + 별%/100)** — 매일 재계산.
- **매수점 = 별지점 − 0.01**, **매도점 = 별지점** (체결 겹침 방지).
- `T ≥ N/2` (후반전)이면 별% 음수 → 별지점이 평단 **아래** → 평단보다 손절을 시도.
- 예) SOXL 20분할, 평단 38.30, T=8.6 → 별%=20−17.2=2.8% → 별지점 = 38.30×1.028 = **39.37$**.

---

## 4. 1회매수금 산출 (V4 — 이전과 다름)

- 첫 매수: `1회매수금 = 원금 / N`.
- 2회차부터: **`1회매수금 = 잔금 / (N − T)`** — 남은 현금을 남은 회차로 분배.
  - 20분할: `잔금/(20−T)`, 40분할: `잔금/(40−T)`.
- T가 +1/+0.5/×0.75로 바뀔 때마다, 잔금도 바뀌므로 매일 미세 변동.
- 예) 원금 2만 / 40분할 → 첫 시도 500. 실제 478 체결 → 잔금 19522, T=1 → `19522/39 = 500.56$`가 2회차 시도액.

---

## 5. 일반모드 — 매수방법 (매일 갱신)

그날 주문 = **★(또는 큰수/평단) 본덩이 + 하단 LOC 사다리(각 1주, 큰 하락 대비 보험)**. 매수는 전부 LOC.

### 5-A. 처음매수 (보유 0, T=0, 평단 없음)
- `큰수 = 현재가 × (1 + 0.10~0.15)` (관행상 +10~15% 위).
- 큰수 LOC에 `floor(1회매수금 / 큰수)` 주 + 하단 사다리 각 1주.
- 예) 현재가 45.935 → 큰수 51.44에 12주(=617.895/51.44) + [47.53/44.13/41.19/38.61/36.34/34.32/32.52] 각 1주.

### 5-B. 전반전 (`0 < T < N/2`, 40분할이면 T=0~19.99…)
- 1회매수금 **절반은 별지점(★) LOC, 절반은 평단 LOC**로 분할.
- + 하단 LOC 사다리 각 1주(평단 아래로).
- 예) 별지점 78.12 → ★(78.11) 3주 + 평단(69.75) 4주 + [67.40/59.91/53.92/49.02/44.93/41.47/38.51] 각 1주.

### 5-C. 후반전 (`N/2 ≤ T ≤ N−1`, 40분할이면 T=20~39)
- 별지점이 평단 아래로 내려옴.
- 1회매수금 **전액을 별지점(★) LOC**.
- + 하단 LOC 사다리 각 1주.
- 예) 별지점 59.55 → ★(59.54) 9주 + [56.85/51.68/47.37/43.73/40.60/37.90/35.53/33.44] 각 1주.

---

## 6. 일반모드 — 매도방법 (전·후반 공통, 매일 갱신)

- **보유 1/4 → 별지점(매도점) LOC 쿼터매도.**
- **보유 3/4(나머지) → 평단 지정가매도.** 지정가 = `평단 × (1 + p)`, `p = 15%(TQQQ) / 20%(SOXL)`.
- 예) 별지점 59.55에 35주(≈보유의 1/4) LOC + 75.76(=평단×1.15)에 106주 지정가. (35+106=141 보유)
- 실거래 권장: 지정가매도는 프리장(서머 22시 / 비서머 23시)부터 걸어 프리·본·애프터 전 구간 활용. 백테스트는 일봉 종가/고가 기준으로 근사(§13).

---

## 7. 일반모드 — 종료시점

- **보유수량 = 0 → 사이클 종료** → 처음매수(§5-A)로 복귀.
- 단, 지정가매도로 보유 0이 된 같은 날 LOC 매수가 체결되면 보유>0 → **종료 아님**, 계속 진행.
- 재시작 자본 정책(택1): **단리**(원금만 재사용, 권장) / **복리**(원금+누적수익). 원금 클수록 변동성에 안정적.

---

## 8. 소진 → 리버스모드 전환

- 발동 조건: **`T ≥ N − 1`** (20분할 T≥19, 40분할 T≥39) = 1회분 미만 남은 시점.
- 남은 잔금도 리버스모드에서 그대로 사용. 모드 = `리버스모드`로 전환.

---

## 9. 리버스모드 — 매도과정

소진 후엔 매도가 메인. divisor `d = N/2` (20분할→10, 40분할→20).

- **첫날: 무조건매도 MOC.** 수량 = `floor(전체보유 / d)`.
- **둘째날 이후: 별지점(매도점) 위 LOC 매도.** 수량 = `floor(직전보유 / d)`, 매일 재계산.
- 예) 40분할 200주: 첫날 floor(200/20)=10 → 잔190 → floor(190/20)=9 → 잔181 → 9 → 잔172 → 8 … (무한매도)
- 매도점 = 별지점(예 48.62).

## 10. 리버스모드 — 매수과정 (쿼터매수)

- **리버스 첫날: 매수 없음** (매도만).
- **둘째날부터: `총잔금 / 4` 금액을 별지점 아래 LOC로 매수시도** (쿼터매수).
  - `총잔금 = 남은 현금잔금 + 직전 매도로 들어온 현금`.
  - 예) 잔금 400 + 직전 MOC 매도금 300 = 700 → `700/4 = 175$`를 둘째날 매수.
- 매수점 = 별지점−0.01(예 48.61) 본덩이 + 하단 LOC 사다리 각 1주 (일반모드 매수표와 모양 동일).
- 매일 총잔금 기준 `잔금/4` 반복.

## 11. 리버스모드 — 종료조건 (일반모드 복귀)

- **종가가 평단 대비 회복선 위로 올라오면 종료.**
  - 회복선 = `평단 × (1 − p)`, `p = 15%(TQQQ) / 20%(SOXL)`.
  - 즉 종가 > 평단×0.85(TQQQ) 또는 종가 > 평단×0.80(SOXL) 확인.
  - (이 −15%/−20%는 일반모드 마지막 T에서의 쿼터매도 기준과 동일.)
  - 예) 평단 40, SOXL 종가 32 초과 → 그날 확인 후 **그 날부터 일반모드** 시작. 하루 만에 종료 가능.
- 리버스 동안은 종가 < 평단 → 매수·매도 모두 손절.
- **복귀 시 T값은 리버스에서 쓰던 값 그대로 이어감.**
- **복귀 시 1회매수금도 일반모드 공식 그대로**: `잔금/(N−T)`.
- 일반모드 복귀 후 진행 중 다시 `T ≥ N−1`(1회분 미만)이면 → **다시 리버스모드** (§8 반복).

## 12. 자금 규칙

- 리버스 진행 중에도 **외부 자금 추가투입 금지**. 시작 시 설정한 원금 내에서 모든 과정 해결.
- 지속 손절이 싫어도 규칙 유지 — 규칙 미준수는 V4.0이 아닌 변형.

## 13. 백테스트 구현 시 결정해야 할 사항

**체결 모델 (일봉 데이터 기준 근사):**
- **LOC 매수**: 종가 ≤ 지정가면 종가로 체결(LOC = Limit On Close). 각 사다리 단가별로 판정.
- **LOC 매도**: 종가 ≥ 매도점이면 종가로 체결.
- **MOC 매도**: 무조건 종가 체결.
- **지정가매도(15%/20%)**: 장중 고가 ≥ 지정가면 지정가로 체결(일봉이면 `high ≥ 지정가` 판정). 분/틱 데이터 있으면 더 정확.
- 같은 날 매수·매도 동시 체결 케이스(§7 종료 예외, §2 T 갱신) 순서 처리 규칙 정의.

**데이터:** TQQQ/SOXL 일봉(시/고/저/종, 분할·배당 조정). 가능하면 분봉으로 지정가 체결 정밀화.

## 14. 정밀 항목 — 언이 계산기로 해소됨 (`uni-calculator-analysis.md` 참조)

원문은 **방법론(원리)**만 주고 단가표 숫자는 계산기가 산출 → 아래 3개가 미확정이었으나, **언이 자동계산 시트 1.1 Apps Script 해부로 해소**(`uni-calculator-analysis.md`). 단, 이는 **언이 변형판** 기준이므로 라오어 순정과 다를 수 있음:

1. **하단 LOC 사다리 간격** → `tierPrice = amt / (baseQty + k)`, k=1,2,… (정수 주수로 1회매수금을 나눈 가격). 고정 %가 아니라 1/k 간격이라 아래로 갈수록 벌어짐.
2. **사다리 단수** → 큰수 미만 조건 만족분만 **최대 5단**.
3. **체결수량 반올림** → 전부 **floor(내림)**.

→ 백테스터는 위 공식을 차용하되, "언이 변형판"임을 인지. 라오어 순정 사다리는 별도 확인 시 교체 가능하도록 파라미터화 권장.

**언이 변형판이 라오어 순정과 다른 주요 지점** (상세 `uni-calculator-analysis.md` §7):
- 별% = 목표수익률×(1−2T/n)로 일반화(목표수익률 커스텀)
- 큰수 = **종가**×(1+10%), 매수가 **상한**으로 작동
- 리버스 기준선 = **5일평균(sma5)** (순정 별지점 대비)
- 잔금에 실현손익 누적 = **복리** 기본
- 리버스 T 갱신식 명시(매수 +=(n−T)×0.25, 쿼터매도 ×(1−2/n))

## 15. 보조 — 분수매수 (방법론 본체와 별개)

- 별지점과 현재가 괴리가 **약 20% 이상**이면 LOC가 매매를 못 갈라줌(한계). 이때 "분수매수" 기법으로 보완.
- V4.0 본 방법론과 무관한 보조 개념. (네이버 카페 글 53077 참조)
- 백테스트 1차 버전에서는 제외(out of scope), 추후 옵션으로 검토.
